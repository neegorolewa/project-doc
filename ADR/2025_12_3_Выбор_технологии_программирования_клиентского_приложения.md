**ADR: 2025-12.3 - Выбор технологии программирования клиентского приложения**

**Дата:** 1 декабря 2025 г.

**Статус:** На проверке

**Контекст:** 

Для создания клиентского приложения системы бронирования отелей нам необходимо выбрать соответствующий фреймворк или библиотеку. От этого выбора зависит пользовательский опыт, скорость загрузки приложения, а также производительность и возможности разработки.

**Рассмотренные варианты:**

1. **React (TypeScript)**: Лидирующая библиотека для построения пользовательских интерфейсов с декларативным подходом и компонентной архитектурой. Имеет крупнейшее сообщество и экосистему.

2. **Vue.js**: Прогрессивный фреймворк, известный простотой интеграции и плавной кривой обучения. Имеет отличную официальную поддержку TypeScript в третьей версии.

3. **Angular**: Полноценный фреймворк с "из коробки" решением для маршрутизации, DI, HTTP-клиента. Строгая архитектура, но более высокий порог входа и меньшая гибкость.

4. **Svelte**: Современный компилируемый фреймворк с высокой производительностью и простым синтаксисом.

**Решение:**

Выбран стек React в связке с TypeScript.

**Обоснование:**

1. **Экосистема и надежность:** React с крупнейшим сообществом обеспечивает скорость разработки, а TypeScript гарантирует безопасность типов и выявление ошибок на этапе создания кода.

2. **Гибкость и производительность:** Архитектура React позволяет гибко выбирать инструменты под задачи, а современные возможности React обеспечивают отзывчивость интерфейсов.

3. **Хорошая интеграция с бэкендом на C#:** Возможность использовать API на ASP.NET Core эффективно.

**Недостатки решения:**

1. **Высокая скорость развития:** Быстрая эволюция экосистемы обязывает постоянно обновлять зависимости и следить за трендами.

2. **Квалификация сотрудников:** Для некоторых команд может потребоваться обучение, если опыт работы с React и TypeScript отсутствует.

**Последствия:**

- **Положительные:**
    1. Ускорение разработки UI за счет компонентного подхода и готовых решений.
    2. Повышение надежности интерфейсов благодаря статической типизации TypeScript.
    3. Возможность параллельной работы команд над публичным клиентом (React/Vite) и панелями управления (Next.js).

- **Отрицательные и нейтральные:**
    1. Потребуется настройка сборки, линтинга и тестовых фреймворков (ESLint).
    2. Потребуется создать и поддерживать общую библиотеку UI-компонентов и строгие стандарты.
